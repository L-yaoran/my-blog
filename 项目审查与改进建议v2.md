## 二、未完成项（仍需改进）

### 2.1 练习页骨架重复（v1 #1.2）

**现状：** 9 个静态 HTML 文件共 6570 行，每个文件各自包含完整的导航栏、工具栏、历史面板、答题详情弹窗。新增第 10 套练习需要编辑全部 9 个已有文件的下拉菜单。

| 文件 | 行数 |
|------|------|
| python_basic_test_001.html | 930 |
| python_basic_test_002.html | 706 |
| python_basic_test_003.html | 755 |
| python_basic_test_004.html | 768 |
| python_basic_test_005.html | 708 |
| python_basic_test_006.html | 825 |
| python_basic_test_007.html | 674 |
| python_basic_test_008.html | 615 |
| python_basic_test_009.html | 589 |
| **合计** | **6570** |

**改进方案：**

| 方案 | 复杂度 | 效果 |
|------|--------|------|
| A. 用 Jinja2 模板渲染全部练习页（推荐） | 中（4-6h） | 骨架只维护一份 `quiz.html`，所有练习页通过 Flask 路由 `/practice/00X/` 访问，JSON 存题目数据 |
| B. 生成脚本模板化 | 低（2-3h） | 保持静态 HTML，但生成脚本共享一个骨架模板，新增练习只需改题目数据 |
| C. 保持现状 | 无 | 每次新增练习手动改 9 个文件的下拉菜单 |

### 2.2 生成脚本未归档（v1 #5.1）

**现状：** 007/008/009 的生成脚本已存在于 `my_blog/scripts/`（`gen_007.py`、`generate_day06.py` 等），但还有 3 个未跟踪的脚本文件。v1 中提到的 `gen007_clean.py`、`gen008.py`、`gen009.py` 实际上在 VS_code 根目录（非 my_blog 内），需要确认是否需要迁入。

**改进方案：**
- 将根目录的 gen 脚本移入 `my_blog/scripts/`
- 删除不再需要的临时脚本（`check_*.py`、`fix_*.py` 等）
- 在 `scripts/` 下添加 README 说明各脚本用途

### 2.3 Flask 路由只覆盖 001（v1 #3.1）

**现状：** `app.py` 的 `/practice/<quiz_id>/` 路由只映射了 `'001': 'quiz_001.json'`。002~009 仍通过 `/static/python_basic_test_00X.html` 静态访问。

**改进方案：**
- 如果采用方案 A（Jinja2 模板渲染），则需要为 002~009 创建 JSON 数据文件，扩展 `quiz_map`
- 如果采用方案 B（保持静态），则无需改动
