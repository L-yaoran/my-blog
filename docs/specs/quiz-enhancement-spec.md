# 练习平台增强 Spec

> 来源：产品规划文档 PRD（2026-06-12 归档）中与练习站相关的功能规划。
> 整理为独立 spec，作为 v4.2（练习平台 MVP）之后的迭代依据。
> 不涉及用户登录、数据库、云端同步——所有功能优先 localStorage 实现。

---

## 1. 背景

当前练习平台已有 16 套练习（570+ 道题），支持选择题/填空题/简答题/代码实操四类题型，具备自动评分、进度暂存、导入导出、深色模式等功能。

但与力扣、牛客等成熟练习平台相比，缺少以下关键能力：

| 缺失能力 | 用户影响 |
|----------|---------|
| 错题收集 | 做错的题分散在各套练习中，无法集中复习 |
| 练习统计 | 不知道自己做了多少题、正确率如何、薄弱点在哪 |
| 题目随机 | 选项顺序固定，容易"背答案"而非真正理解 |
| 难度标记 | 所有题目看起来一样，无法区分基础和进阶 |
| 代码运行 | 代码题只能看参考答案，不能在浏览器中运行验证 |
| 计时功能 | 无法模拟考试场景 |

---

## 2. 功能清单

### 2.1 错题本（P0）

**目标：** 提交后自动收集错题，支持集中复习和重新作答。

**数据结构：**

```javascript
// localStorage key: 'quiz_wrong_answers'
{
  "quiz_001": {
    "qid_3": {
      "question": "题目文本...",
      "user_answer": "A",
      "correct_answer": "B",
      "explanation": "解析...",
      "timestamp": "2026-06-12T10:30:00",
      "reviewed": false
    }
  }
}
```

**功能要求：**

- 每次提交后自动将错题写入 `quiz_wrong_answers`
- 独立错题本页面或面板，支持按练习编号筛选
- 每道错题显示：题目、用户答案、正确答案、解析
- 支持"标记已掌握"（从错题本移除）
- 支持"重新作答"（跳转回原题或在错题本内作答）
- 支持错题数据导出（JSON）

**涉及文件：**

- `static/js/quiz-engine.js` — 提交后自动收集错题
- 新增 `static/js/wrong-book.js` — 错题本页面逻辑
- 新增 `static/wrong-book.html` — 错题本页面

**验收标准：**

- [ ] 提交后错题自动保存
- [ ] 错题本页面可查看所有错题
- [ ] 支持按练习编号筛选
- [ ] 支持标记已掌握
- [ ] 支持导出 JSON
- [ ] 深色模式正常

---

### 2.2 练习统计面板（P1）

**目标：** 显示总做题数、正确率、各题型分布、每日练习趋势。

**数据来源：** 现有 localStorage 中的 `quiz_history` 数据，无需新增存储。

**统计指标：**

| 指标 | 计算方式 |
|------|---------|
| 总做题次数 | 所有 quiz_history 的提交记录数 |
| 总做题套数 | 有多少套练习至少提交过一次 |
| 客观题正确率 | 所有客观题得分 / 所有客观题总分 |
| 各题型分布 | 选择题/填空题/简答题/代码题各做了多少 |
| 每日练习趋势 | 最近 7/30 天的提交次数折线图 |
| 最薄弱知识点 | 正确率最低的 3 套练习 |

**展示方式：**

- 独立统计页面或首页可折叠面板
- 数字指标 + 简单图表（CSS 柱状图或 SVG 折线图）
- 不引入 Chart.js 等重依赖

**涉及文件：**

- 新增 `static/js/stats.js` — 统计计算逻辑
- 新增 `static/stats.html` — 统计页面

**验收标准：**

- [ ] 显示总做题次数和套数
- [ ] 显示客观题正确率
- [ ] 显示最近 7 天练习趋势
- [ ] 深色模式正常

---

### 2.3 题目随机顺序（P1）

**目标：** 打乱选择题选项顺序，防止背答案。

**实现方案：**

- 在 `quiz-engine.js` 的 `DOMContentLoaded` 中，遍历所有选择题卡片
- 根据 `STORAGE_KEY` + 当天日期生成 seed（同一天顺序相同，第二天变化）
- Fisher-Yates shuffle 打乱选项 `<label>` 元素
- `data-answer` 属性随选项移动更新（A→C, B→A 等）
- 填空题/简答题/代码题不受影响

**注意事项：**

- 打乱后 `data-answer` 必须正确映射，否则评分出错
- 已有做题记录的恢复功能需兼容随机顺序
- 首次加载时打乱，暂存/加载时保持已打乱的顺序

**涉及文件：**

- `static/js/quiz-engine.js` — 新增 `shuffleOptions()` 函数

**验收标准：**

- [ ] 每次加载页面选项顺序不同（同一天可相同）
- [ ] 评分结果正确
- [ ] 不影响填空题/简答题/代码题
- [ ] 暂存/加载兼容

---

### 2.4 难度分级（P2）

**目标：** 每道题标记难度，帮助用户选择性练习。

**难度等级：**

| 等级 | 标签色 | 说明 |
|------|--------|------|
| 基础 | 绿色 | 概念记忆、基本语法 |
| 进阶 | 金色 | 综合运用、场景分析 |
| 综合 | 红色 | 跨知识点、实战设计 |

**数据格式变更：**

```python
# quiz_data_XXX.py 中每道题新增 difficulty 字段
{
    'qid': 1,
    'question': '...',
    'options': [...],
    'answer': 'B',
    'analysis': '...',
    'difficulty': 'basic'  # basic / advanced / comprehensive
}
```

**展示方式：**

- 题目卡片右上角显示难度标签
- 支持按难度筛选（顶部筛选栏）
- 统计面板可按难度维度统计正确率

**涉及文件：**

- `scripts/quiz_data_*.py` — 每道题添加 difficulty 字段
- `scripts/generate_quiz.py` — 生成时输出难度标签 HTML
- `static/js/quiz-engine.js` — 筛选逻辑

**验收标准：**

- [ ] 每道题显示难度标签
- [ ] 支持按难度筛选
- [ ] 无 difficulty 字段的旧题默认标记为基础

---

### 2.5 代码题在线运行（P2）

**目标：** 代码实战题支持在浏览器内运行 Python 代码。

**技术方案：** Pyodide（WebAssembly 版 CPython）

**实现要点：**

- 加载 Pyodide runtime（CDN 或本地，约 15MB gzipped）
- 代码题新增"运行"按钮
- 运行结果显示在代码编辑器下方
- 支持 stdout/stderr 输出
- 超时限制（10 秒）
- 沙箱安全（无文件系统、无网络）

**涉及文件：**

- `static/js/quiz-engine.js` — 新增 `runCode()` 函数
- `scripts/quiz_template.html` — 代码题卡片新增运行按钮

**验收标准：**

- [ ] 点击"运行"按钮执行代码
- [ ] 显示 stdout 输出
- [ ] 显示 stderr 错误信息
- [ ] 超时自动终止
- [ ] 移动端可用

---

### 2.6 计时模式（P2）

**目标：** 可选开启计时，模拟考试场景。

**功能要求：**

- 练习页顶部工具栏新增"计时模式"开关
- 开启后显示倒计时或正计时
- 可设置时长（30 分钟 / 60 分钟 / 90 分钟 / 不限）
- 时间到自动提交
- 提交记录中包含用时信息

**涉及文件：**

- `static/js/quiz-engine.js` — 新增计时逻辑
- `scripts/quiz_template.html` — 工具栏新增计时控件

**验收标准：**

- [ ] 可开启/关闭计时模式
- [ ] 显示已用时间
- [ ] 可设置时间限制
- [ ] 时间到自动提交

---

### 2.7 数据导入合并增强（P2）

**目标：** 导入 JSON 时合并已有进度，而非覆盖。

**当前行为：** `importProgress()` 直接用导入数据覆盖 localStorage。

**目标行为：**

- 导入前显示预览（将导入多少条记录）
- 提供选项：合并 / 覆盖
- 合并模式：保留已有记录，追加新记录（按时间戳去重）
- 覆盖模式：保持现有行为

**涉及文件：**

- `static/js/quiz-engine.js` — 修改 `importProgress()` 函数

**验收标准：**

- [ ] 导入前显示预览
- [ ] 支持合并和覆盖两种模式
- [ ] 合并模式按时间戳去重

## 3. 实施优先级

| 阶段 | 功能 | 预估工时 |
|------|------|---------|
| **Phase A** | 错题本 | 4-6h |
| **Phase A** | 题目随机顺序 | 2h |
| **Phase B** | 练习统计面板 | 3-4h |
| **Phase B** | 难度分级 | 3-4h（含数据标注） |
| **Phase C** | 数据导入合并增强 | 2h |
| **Phase C** | 计时模式 | 2-3h |
| **Phase D** | 代码题在线运行 | 8-10h |

Phase A 可在 v4.2 之前独立实施（不依赖练习平台首页）。
Phase B/C/D 建议在 v4.2 完成后实施。

---

## 4. 不做的事（来自 PRD 约束）

| 不做 | 原因 |
|------|------|
| 云端进度同步 | 需要用户登录 + 数据库，当前架构不支持 |
| 学习报告自动生成 | 依赖云端数据，localStorage 不适合长期分析 |
| 社区讨论功能 | 可通过 GitHub Issues 替代 |
| AI 自动出题 | 超出个人项目范围 |
| 移动端 App | 响应式网页已够用 |
