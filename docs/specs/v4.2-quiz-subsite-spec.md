# v4.2 Spec：Quiz 子站 / 练习平台 MVP

> 阶段定位：将当前“散落在主站首页和 `/static/` 下的练习页”升级为一个有独立入口、清晰分类和可扩展结构的练习平台 MVP。  
> 推荐目标域名：`quiz.829007.xyz`。  
> 本阶段不做用户登录、不做数据库、不做服务器保存进度。

---

## 1. 背景与目标

### 1.1 当前状态

当前练习系统已经具备：

- 16 套静态练习页。
- 统一生成脚本 `scripts/generate_quiz.py`。
- 统一题库数据 `scripts/quiz_data_0XX.py`。
- 统一元数据 `scripts/quiz_meta.py`。
- 统一模板 `scripts/quiz_template.html`。
- 统一前端引擎 `static/js/quiz-engine.js`。
- 支持：选择题、填空题、简答题、代码实战、评分、做题记录、暂存、导入导出、CodeMirror。

当前主要问题：

1. 练习入口混在首页大量卡片中。
2. URL 是 `/static/python_basic_test_016.html`，不像正式产品路径。
3. 首页维护大量练习卡片，后续会越来越臃肿。
4. 练习没有独立目录页、阶段分类、搜索筛选。
5. 主站教程与练习之间还没有系统联动。

### 1.2 阶段目标

v4.2 的目标是：

> 将练习功能产品化为“LLM 交互练习平台 MVP”，主站保留入口，练习平台拥有独立首页、清晰分类和可映射子域名的结构。

---

## 2. 成功标准

完成后应满足：

- 主站首页不再铺开 16 个练习卡片。
- 有独立练习平台首页。
- 练习按阶段 / 主题分类。
- 每套练习有清晰标题、题量、知识点、入口。
- 旧练习 URL 继续可访问。
- 新练习入口 URL 更友好。
- 可配置 `quiz.829007.xyz` 指向练习平台。
- sitemap 包含练习平台入口和练习页。
- 不破坏现有 localStorage 存档。

---

## 3. 推荐方案

## 3.1 三种方案判断

### 方案 A：继续全部放主站

不推荐长期使用。简单但首页和主站会继续膨胀。

### 方案 B：同仓拆分 quiz 模块

推荐。

优点：

- 仍是一个仓库，适合个人维护。
- 主站和练习平台边界清晰。
- 可映射子域名。
- 不需要立刻引入复杂前端框架。

### 方案 C：完全独立仓库

暂不推荐。当前还没有用户系统、题库后台、商业化，不值得增加维护成本。

## 3.2 最终推荐

采用：

> **同仓 quiz 模块 + Nginx 子域名映射 + 旧 URL 兼容。**

## 5. 路由与 URL 设计

## 5.1 主站路由

新增：

```text
/quiz/
```

作用：练习平台首页。

可选新增：

```text
/quiz/<quiz_id>/
```

短期可以 302 跳转到旧静态页：

```text
/quiz/016/ → /static/python_basic_test_016.html
```

## 5.2 子域名 URL

Nginx 层配置：

```text
quiz.829007.xyz/           → 练习平台首页
quiz.829007.xyz/016/       → 练习 016
quiz.829007.xyz/static/... → 静态资源
```

### 短期映射方式

可让 `quiz.829007.xyz` 仍反代同一个 Flask 应用，但默认进入 `/quiz/`。

### 旧 URL 兼容

必须保留：

```text
https://829007.xyz/static/python_basic_test_001.html
...
https://829007.xyz/static/python_basic_test_016.html
```

原因：

- 已经可能被用户收藏。
- localStorage key 不依赖 URL，但用户习惯和外链依赖旧地址。
- 避免 sitemap 和搜索引擎断链。

## 13. localStorage 兼容要求

不能修改已有 key：

```text
STORAGE_KEY
SAVE_KEY
```

原因：

- 用户做题记录依赖这些 key。
- 改 URL 不应导致数据丢失。

如果未来迁移 key，必须提供迁移脚本：

```javascript
old_key → new_key
```

v4.2 不做 key 迁移。

## 14. 风险与处理

### 风险 1：子域名代理路径错误

处理：先让 `quiz.829007.xyz` 指向 `/quiz/` 首页，再逐步处理 `/016/`。

### 风险 2：旧 URL 断链

处理：保留 `/static/python_basic_test_XXX.html`。

### 风险 3：首页删除练习卡片后用户找不到旧练习

处理：练习平台首页必须列出全部练习。

### 风险 4：元数据重复维护

处理：以 `quiz_meta.py` 作为唯一来源，`catalog.json` 由脚本生成。

### 风险 5：Nginx / SSL 配置不在仓库

处理：单独记录部署步骤，改动前备份服务器 Nginx 配置。
