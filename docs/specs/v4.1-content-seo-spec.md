# v4.1 Spec：内容系统、分类标签、SEO、RSS 与搜索

> 阶段定位：在 v4 首页定位清晰后，把博客从“少量 Markdown 文章列表”升级为可长期扩展的内容系统。  
> 本阶段仍然保持 Flask + Flask-FlatPages，不引入数据库，不迁移框架。

---

## 1. 背景与目标

### 1.1 当前状态

当前内容系统特点：

- 文章存放在 `content/`。
- 使用 Flask-FlatPages 读取 Markdown。
- `app.py` 通过 `get_posts()` 只筛选包含 `date` 的页面并按日期排序。
- 文章 frontmatter 目前较简单，常见字段为：

```yaml
title: xxx
date: 2026-05-31
badge: 项目实践
summary: xxx
```

当前缺少：

- 分类页。
- 标签页。
- 教程页。
- 项目页。
- 工具页。
- RSS。
- 动态 sitemap。
- Open Graph。
- 搜索索引。
- 系列教程字段。

### 1.2 阶段目标

v4.1 的目标是：

> 建立一个轻量但结构清晰的内容系统，让博客、笔记、教程、项目、工具和练习可以被分类、检索、关联和 SEO 收录。

---

## 2. 成功标准

完成后应满足：

- Markdown 内容支持统一 frontmatter。
- 有独立 `/blog/` 页面。
- 有独立 `/tutorials/` 页面。
- 有独立 `/projects/` 页面。
- 有独立 `/tools/` 页面。
- 文章页输出 SEO meta、description、Open Graph。
- `/sitemap.xml` 动态生成并包含最新文章和核心页面。
- `/rss.xml` 可访问。
- 可生成 `static/search-index.json`。
- 测试覆盖核心路由。

## 3. 内容模型设计

## 3.1 统一 frontmatter 字段

建议所有内容文件使用以下字段：

```yaml
title: 文章标题
date: 2026-06-12
updated: 2026-06-12
summary: 一句话摘要
category: AI / LLM
type: blog
tags:
  - LLM
  - Prompt
  - RAG
series: LLM 入门教程
series_order: 1
difficulty: beginner
featured: true
cover: /static/images/covers/example.png
status: published
related_quizzes:
  - "013"
  - "014"
related_projects:
  - llm-quiz-platform
```

### 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | 是 | 页面标题 |
| `date` | 是 | 发布日期 |
| `updated` | 否 | 更新日期 |
| `summary` | 是 | 卡片摘要和 meta description |
| `category` | 是 | 内容分类 |
| `type` | 是 | `blog` / `note` / `tutorial` / `project` / `tool` |
| `tags` | 否 | 标签数组 |
| `series` | 否 | 系列名称 |
| `series_order` | 否 | 系列顺序 |
| `difficulty` | 否 | `beginner` / `intermediate` / `advanced` |
| `featured` | 否 | 是否首页精选 |
| `cover` | 否 | 封面图 |
| `status` | 否 | `draft` / `published` |
| `related_quizzes` | 否 | 关联练习编号 |
| `related_projects` | 否 | 关联项目 slug |

## 3.2 内容类型定义

### blog

技术博客、项目复盘、工具评测。

### note

学习笔记、阶段总结、踩坑记录。

### tutorial

系统性教程，适合加入学习路线。

### project

项目 case study。

### tool

工具或资源入口说明页。

## 4. 路由设计

## 4.1 新增路由

在 `app.py` 中新增：

```text
/blog/
/tutorials/
/projects/
/tools/
/tags/<tag>/
/categories/<category>/
/rss.xml
```

现有路由保留：

```text
/
/about
/<path:slug>/
/robots.txt
/sitemap.xml
```

## 4.2 路由职责

### `/blog/`

显示：

- type 为 `blog` 或未指定但有 date 的文章。
- 支持按日期倒序。

### `/tutorials/`

显示：

- type 为 `tutorial`。
- 按 `series` 和 `series_order` 分组。

### `/projects/`

显示：

- type 为 `project`。
- featured 项目优先。

### `/tools/`

显示：

- type 为 `tool`。
- 可以混合静态工具卡片和 Markdown 工具页。

### `/tags/<tag>/`

显示包含该 tag 的所有内容。

### `/categories/<category>/`

显示该分类下所有内容。

## 6. SEO Spec

## 6.1 base.html 新增 block

在 `templates/base.html` 增加：

```jinja2
{% block meta_description %}
<meta name="description" content="AI幻世录·GeoLib：记录 AI / LLM 学习、项目实践、交互练习和工具资源。">
{% endblock %}

{% block og_meta %}
<meta property="og:title" content="AI幻世录·GeoLib">
<meta property="og:description" content="AI / LLM 学习型作品集网站。">
<meta property="og:type" content="website">
<meta property="og:url" content="https://829007.xyz/">
{% endblock %}
```

文章页覆盖：

```jinja2
{% block meta_description %}
<meta name="description" content="{{ post.meta.summary }}">
{% endblock %}
```

## 6.2 文章页 SEO

`post.html` 应输出：

```text
<title>{{ title }} — AI幻世录·GeoLib</title>
meta description
og:title
og:description
og:type=article
article:published_time
article:modified_time
article:tag
canonical
```

## 6.3 JSON-LD

文章页可加入：

```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "文章标题",
  "datePublished": "2026-06-12",
  "dateModified": "2026-06-12",
  "author": {
    "@type": "Person",
    "name": "明屿 Geoff"
  }
}
```

项目页可使用：

```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication"
}
```

## 9. 搜索 Spec

## 9.1 MVP 搜索方式

不引入后端搜索服务。使用静态 JSON + 前端搜索。

生成：

```text
static/search-index.json
```

### 数据结构

```json
[
  {
    "title": "从零构建交互式 LLM 练习平台",
    "url": "/interactive-python-quiz-platform/",
    "type": "blog",
    "category": "项目实践",
    "tags": ["Flask", "JavaScript", "LLM"],
    "summary": "...",
    "content": "正文纯文本，可截断"
  }
]
```

## 9.2 生成脚本

新增：

```text
scripts/build_search_index.py
```

功能：

- 遍历 `content/`。
- 读取 frontmatter。
- 提取正文纯文本。
- 输出 JSON。

## 9.3 前端搜索

两种方案：

### 方案 A：原生 JS 简单搜索

适合文章少于 100 篇。

### 方案 B：Fuse.js

适合模糊匹配。

v4.1 可以先用方案 A。

## 13. 风险与处理

### 风险 1：旧文章缺少字段导致模板报错

处理：所有模板使用 `.get()` 默认值。

### 风险 2：中文 tag URL 编码问题

处理：先使用英文 slug tag，中文展示名后续再做映射。

### 风险 3：动态 sitemap 和静态 sitemap 冲突

处理：确认 `app.py` `/sitemap.xml` 路由优先返回动态内容。

### 风险 4：FlatPages 对复杂 frontmatter 支持有限

处理：先测试 tags 数组是否能正常读取；如不稳定，可采用逗号字符串：

```yaml
tags: LLM,Prompt,RAG
```

并在 Python 中拆分。
