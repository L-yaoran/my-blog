# v4 Spec：产品定位、信息架构与首页改版

> 阶段定位：把 `my_blog` 从“博客 + 练习堆叠页”升级为清晰的 AI / LLM 学习型作品集首页。  
> 本阶段只优化主站结构和首页表达，不拆 quiz 子站，不接入 AI 助手。

---

## 1. 背景与目标

### 1.1 当前问题

当前项目已经具备：

- Flask + Flask-FlatPages 博客系统。
- 首页 `templates/hub.html` 集中展示博客、练习、项目、关于。
- 16 套静态交互练习页。
- 基础深色模式、响应式、导航和部署能力。

但首页目前主要问题是：

1. 首屏更像个人宣言，不够明确表达“AI / LLM 作品集”定位。
2. 导航只有首页、关于、GitHub，不适合长期承载博客、教程、项目、练习、工具。
3. 练习卡片太多，占据首页主要空间。
4. 项目展示不足，不利于求职展示。
5. 首页内容全部硬编码在 `hub.html`，维护成本逐渐变高。

### 1.2 阶段目标

v4 的目标是：

> 让访客进入首页后 5 秒内理解：你是谁、你在做什么方向、这个站点能提供什么、有哪些代表项目。

### 1.3 非目标

v4 不做：

- 不拆分 `quiz.829007.xyz`。
- 不改练习平台核心逻辑。
- 不接入 Claude API / AI 助手。
- 不引入数据库。
- 不整体迁移 Next.js / Astro。
- 不做用户登录。

---

## 2. 成功标准

完成后应满足：

- 首页 Hero 明确表达“AI / LLM 学习型作品集”定位。
- 顶部导航包含博客、教程、项目、练习、工具、关于、GitHub。
- 首页不再完整铺开 16 个练习卡片，只保留练习平台入口和少量精选。
- 项目区至少展示 2～3 个代表项目。
- 首页有“LLM 学习路线”模块。
- 首页有“工具与资源”入口。
- 移动端布局正常。
- 深色模式正常。
- 不破坏现有文章、练习页访问。

## 9. 风险与回滚

### 风险 1：首页改动影响现有练习入口

处理：保留至少一个明显的“进入练习平台”按钮。

### 风险 2：hub.html 改动过大导致样式混乱

处理：先复制当前结构，分模块替换，不一次性重写全部。

### 风险 3：深色模式漏样式

处理：每新增一个模块，同步写 `.dark-mode` 样式。

### 回滚方式

如果是 Git 仓库：

```bash
git checkout -- templates/base.html templates/hub.html static/css/style.css static/js/main.js
```
